# Trading System

整合 LPPLS + ICT + PA + 归纳演绎验证 的加密货币交易分析系统

## 核心分析流程（五步验证法）

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Step 1: LPPLS泡沫模型    →  宏观判断：是否有泡沫？风险等级？          │
│  Step 2: ICT结构分析      →  中观判断：机构资金流向？结构转换？        │
│  Step 3: PA形态确认      →  微观确认：K线形态是否有效？               │
│  Step 4: 归纳法验证      →  历史数据验证：历史相似走势统计           │
│  Step 5: 演绎法+二阶验证  →  理论推导 + 多维度交叉确认               │
└─────────────────────────────────────────────────────────────────────────┘
```

## Step 1: LPPLS 泡沫模型（宏观判断）

### 目的
判断市场是否处于泡沫状态，识别崩盘风险。

### LPPLS 核心公式
```
P(t) = A + B(t_c - t)^m + C(t_c - t)^m * cos(ω*ln(t_c - t) + φ)
```

### 关键参数
| 参数 | 含义 | 风险判断 |
|------|------|----------|
| m | 稳定性指数 (0<m<1) | 接近1=加速泡沫，接近0=趋于稳定 |
| ω (omega) | 振荡频率 (4-13) | >10 = 高波动风险 |
| ψ (psi) | 泡沫指数 | <-0.5 强泡沫，<-0.3 中度，<-0.1 轻度 |

### 判断规则
```
if ψ < -0.5:
    风险等级 = "高风险泡沫"
    建议 = "减仓/观望，等待LPPLS信号消退"
elif ψ < -0.3:
    风险等级 = "中度泡沫"
    建议 = "谨慎做多，设置硬止损"
elif ψ < -0.1:
    风险等级 = "轻度泡沫"
    建议 = "正常交易，但需监控"
else:
    风险等级 = "正常"
    建议 = "无泡沫风险"
```

### 数据要求
- 至少60-100个交易日数据
- 日收盘价

### 输出格式
```
【LPPLS 泡沫分析】
- 风险等级: 高/中/低/正常
- m值: X.X (临界状态/稳定)
- ω值: X.X (高频振荡/正常)
- ψ指数: X.XXX
- 建议: [具体操作建议]
```

## Step 2: ICT 结构分析（中观判断）

### 核心概念

#### 2.1 Order Block (OB) - 订单区块
- **看涨OB**: 下跌后的绿色K线，机构买入区
- **看跌OB**: 上涨后的红色K线，机构卖出区
- **特征**: 放量 + 长影线 + 盘整

#### 2.2 Fair Value Gap (FVG) - 公平价值缺口
- **看涨FVG**: Low[i-1] > High[i+1] → 回测买入机会
- **看跌FVG**: High[i-1] < Low[i+1] → 回测卖出机会

#### 2.3 Liquidity Pool - 流动性池
- **Highs**: 前期高点（上方流动性）
- **Lows**: 前期低点（下方流动性）

#### 2.4 Market Structure Shift (MSS) - 市场结构转换
- **看涨MSS**: 突破更高高点(HH) + 不破前低(HL)
- **看跌MSS**: 跌破更低低点(LL) + 不破前高(LH)

#### 2.5 Break of Structure (BOS) - 结构突破
- 回调被突破 = 趋势延续确认

### 判断流程
```
1. 识别当前趋势: 上涨/下跌/震荡
2. 寻找OB: 最近2-5根K线内的订单区块
3. 寻找FVG: 3根K线组合的缺口
4. 定位流动性池: 前后高低价
5. 确认MSS: 结构转换是否发生
6. 确认BOS: 趋势延续/反转
```

### 输出格式
```
【ICT 结构分析】
- 当前趋势: 上涨/下跌/震荡
- 看涨OB: [位置, 强度]
- 看跌OB: [位置, 强度]
- FVG: [看涨/看跌, 位置]
- 流动性池: [上方/下方]
- MSS: [已确认/未确认, 方向]
- BOS: [已确认/未确认]
- 信号: 做多/做空/观望
```

## Step 4: 归纳法验证（历史数据验证）

### 目的
用历史数据总结规律，验证当前走势与历史的相似性。

### 方法

#### 4.1 相似走势匹配
```
1. 提取最近N天价格序列
2. 在历史数据中寻找相似序列（相关系数 > 0.8）
3. 统计相似走势后的涨跌概率
4. 输出统计结果
```

#### 4.2 统计规律总结
```
历史验证:
- 相同LPPLS风险等级下的后续走势: 70%上涨/30%下跌
- 相同ICT信号后的成功率: 65%
- 相同PA形态后的成功率: 72%
```

### 输出格式
```
【归纳法验证】
- 相似历史样本: N个
- 后续上涨概率: XX%
- 后续下跌概率: XX%
- 平均涨幅: X.X%
- 平均跌幅: -X.X%
- 结论: [统计支撑/不支持] 当前交易方向
```

## Step 5: 演绎法 + 二阶验证（理论推导 + 交叉确认）

### 5.1 演绎法 - 理论推导

#### 逻辑链条
```
IF (LPPLS显示泡沫) AND (ICT显示MSS) AND (PA显示顶反)
THEN: 强烈看空（多重理论支撑）

IF (LPPLS正常) AND (ICT显示上涨结构) AND (PA显示底反)
THEN: 强烈看多（多重理论支撑）
```

#### 演绎规则
```
1. 大前提: 泡沫+顶反形态 = 高概率下跌
2. 小前提: 当前LPPLS=高风险, PA=黄昏之星
3. 结论: 应该做空/减仓
```

### 5.2 二阶验证 - 多维度交叉确认

#### 验证维度
| 维度 | 指标 | 权重 |
|------|------|------|
| LPPLS | 泡沫指数 | 30% |
| ICT | 结构信号 | 25% |
| PA | 形态信号 | 20% |
| 成交量 | 量价配合 | 15% |
| RSI | 超买超卖 | 10% |

#### 评分规则
```
看多总分 = LPPLS(30) + ICT(25) + PA(20) + 成交量(15) + RSI(10)
看空总分 = 逆向计算

if 看多总分 > 70: 强烈做多
elif 看多总分 > 50: 轻度做多
elif 看空总分 > 70: 强烈做空
elif 看空总分 > 50: 轻度做空
else: 观望
```

### 输出格式
```
【演绎法 + 二阶验证】
- 逻辑推导: [理论推导结论]
- LPPLS权重得分: X/30
- ICT权重得分: X/25
- PA权重得分: X/20
- 成交量权重得分: X/15
- RSI权重得分: X/10
- 看多总分: XX/100
- 看空总分: XX/100
- 最终信号: 强烈做多/轻度做多/观望/轻度做空/强烈做空
```
